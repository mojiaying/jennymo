# JS执行环境

JS历来是在浏览器里运行的语言，后来发展为可以桌面执行或在服务器上执行， 可以通过javascript shell 或者在vscode中安装run code 插件实现。

# 声明与初始化变量

什么叫变量提升：
Js在执行代码之前会从上到下把代码分析一遍， 即是代码预解析，
在代码预解析时对所有var和function声明的变量进行变量提升，被提升的变量可以在声明之前使用 var let/const 的区别：

名称 var let / const
变量提升 创建和初始化都被提升， 可以重复声明 创建过程被提升，初始化没有提升， 不能重复声明
预解析与执行过程 预解析时创建 ，初始化为undefined，执行时赋值为声明的值 预解析时创建，执行是初始化为声明的值
声明前使用 undefined 块级顶部形成临时行死区， 声明前使用会报错
作用域 函数作用域 块级作用域
全局声明挂载位置 声明的全局变量挂在window下 声明的全局变量挂载在块级作用域script下面
  • 对未声明的变量初始化，该变量就是全局变量
  • let 和 const 唯一区别 const 只有「创建」和「初始化」，没有「赋值」过程，const初始化后修改会报错（但引用类型的值可以改变）， let在初始化之后改变变量值即是赋值过程。
  • const声明的引用类型的值可以用Object.freeze(obj) 冻结变量
  • function 声明变量 ，「创建」「初始化」和「赋值」都被提升了，function 声明的变量可以在声明之前正常使用
  • this声明变量多用于函数内部，它永远指向调用他的那个对象， this声明的变量是作用于上下文的。

# 判断

  • if 一般用于连续区间值判断，复杂的多条件判断，或者动态表达式判断
  • switch 一般用于单一条件的判断
switch(num){
  // 没有break不会中断
	case 1:
	case 2:

}

# 循环结构

# 循环结构的执行步骤

  1. 声明循环变量；
  2. 判断循环条件;
  3. 执行循环体；
  4. 更新循环变量；
  5. 然后循环执行2-4，直到条件不成立，跳出循环。

# 环控制语句

  1. break：跳出本层循环，继续执行循环后面的语句。
    如果循环有多层，则break只能跳出一层。
  2. continue：跳过本次循环剩余的代码，继续执行下一次循环。 ①对与for循环，continue之后执行的语句，是循环变量更新语句(i++)； ②对于while、do-while循环，continue之后执行的语句，是循环条件判断； 　因此，使用这两个循环时，必须将continue放到循环变量更新语句(i++)之后使用，否则，会跳过循环变量更新语句(i++)进入死循环。

# 作用域

Js代码执行前，浏览器会给一个全局作用域window，Window分两个模块一个是存储模块一个是执行模块
存储模块：找到所有的var和function 关键字给这些变量添加内存地址
执行模块：代码从上到下执行，遇到变量就会去存储模块查找，有和没有
有就使用，没有就继续向父级作用域查找直到window下结束，这种查找机制叫作用域链

eg1:函数内部作用域

var a = 10
function fn() {
  console.log(a)
}
function show(f) {
  var a =  20
  f()
}
show(fn)

执行逻辑: 在全局执行环境里先执行for循环，for循环会执行5次，每次都会执行setTimeout函数
由于JavaScript是单线程的，只有一个执行栈,所以会先把setTimeout函数先放进一个任务队列中, 等到执行栈中执行完了之后再完成任务队列中的任务,
for循环执行完了之后i=5, 再执行5个setTimeout, console.log(i)处于全局执行环境,所以每个setTimeout打印出来的都是4
解决：

  1. 使用let声明变量
  2. 使用闭包
for(var a = 0; a < 5; a++) {
  (function(x) {
    setTimeout(function(){
    console.log(x)
  }, 1000)
  })(i)
}

# 递归

递归：函数将自身作为子例程调用 递归的思想是把一个大型复杂问题层层转化为一个与原问题规模更小的问题，问题被拆解成子问题后，递归调用继续进行，直到子问题无需进一步递归就可以解决的地步为止。

为了确保递归正确工作，递归程序应该包含2个属性：

  1. 基本情况（bottom cases），基本情况用于保证程序调用及时返回，不在继续递归，保证了程序可终止。
  2. 递推关系（recurrentce relation），可将所有其他情况拆分到基本案例。

特点：

  1. 优雅性：大大减少了程序的代码量，而且很好理解。递归的能力在于用有限的语句来定义对象的无限集合。
  2. 反向性：由于递归调用程序需要维护调用栈，而栈（我们在上文提过）具有后进先出的特征，因此递归程序适合满足取反类需求。 什么时候考虑递归
  3. 递推关系：递归程序可以较明显的发现递推关系，反过来也可以这么说，具有递推关系的问题基本都可以通过递归求解

抓住问题本身是否可以通过层层拆解到最小粒度来得解

  this.deposit = deposit // 方法
  this.withdraw = withdraw // 方法
  this.toString = toString // 方法
}

function deposit(amount) {
  this.balance += amount
}
funciton withdraw(amount) {
  amount <= this.balance ? this.balance -= amount : alert('Insufficient funds')
}
function toString(amount) {
  return "Balance: " + this.balance
