# 变量声明

编程语言的本质就是用合适的数据结构来存储数据，用合适的算法来管理数据 JS中用不同类型的变量来存储数据，生命变量关键字有： var , let , const

# 弱类型

在JS中变量类型由所引用的值决定, 引用的值类型改变则类型改变

# 变量提升

Js在执行代码之前会从上到下把代码解析一遍， 即是代码预解析，在代码预解析时对所有var声明的变量提升到最前

// 案例一： while不能做为变量名，没有到执行环节就出错了，因为解析代码时声明提前
var web = 'jenny';
console.log(web);
let while = 'aaa'; // Uncaught SyntaxError: Unexpected token 'while'
// 案例二： 在赋值之前使用值为undefinded
var web = "jenny";
function test() {
  if (false) {
    var web = "Tron";
  }
  console.log(web); // undefined (因为在函数作用域内会重新声明)
}
test();
// 案例三： let / const 声明变量在声明之前使用会报错
console.log(x); // Cannot access 'x' before initialization
let x = 1;
// 案例三：变量没有声明直接使用会报错
function test(a = b, b = 3) {} //Cannot access 'b' before initialization
// 案例四：已经赋值的变量可以使用
function hd(a = 2, b = a) {}

  1. 块内存在 let 或者 const 命令， 这个块就会成为一个封闭的作用域， a. 用 let 和 const 声明的全局变量并没有在全局对象中（window），只是一个块级作用域（Script）中 b. 变量在「创建」过程被提升了，但是初始化没有提升，同一个块中不能重复声明 c. 如果在声明之前就开始使用，它并不会引用外部的变量，块级顶部形成临时性死区（TDZ），使用该变量都会被报错。 d. 代码执行时初始化，let 和 const 唯一区别 const 只有「创建」和「初始化」，没有「赋值」过程。let在初始化之后对变量继续操作是赋值
  2. var 的「创建」和「初始化」都被提升了（初始化为undefined）。在执行代码时进行赋值
  3. function 的「创建」「初始化」和「赋值」都被提升了。可以在声明之前使用

# TDZ

TDZ 又称暂时性死区，指变量在作用域内已经存在，但必须在let/const声明后才可以使用。

TDZ可以让程序保持先声明后使用的习惯，让程序更稳定。

建议使用let/const 而少使用var

使用let/const 声明的变量在声明前存在临时性死区（TDZ）使用会发生错误

问题： A. 声明全局变量挂在位置区别 B. 预解析和执行区别（ 创建，初始化和赋值） C. 重复声明区别 D. 声明之前使用区别 E. let 和const 区别：有赋值过程，可重新赋值

const:同一个块中只能声明一次，必须马上初始化，没有赋值过程， 修改引用类型变量的值

# 作用域

函数作用域：
函数中声明的变量就像声明了私有领地，外部无法访问
使用 var 声明的变量存在于最近的函数或全局作用域中，没有块级作用域的机制。

块级作用域： 每一个块{}都是独立作用域，里层作用域可以声明外层作用域同名变量，但不会改变外层变量 使用 let/const 声明的变量, 有块级作用域，只在块域中有效

以往没有块任用时使用立即执行函数模拟块作用域

# Object.freeze

如果冻结变量后，变量也不可以修改了，使用严格模式会报出错误。

"use strict"
const INFO = {
  url: 'https://www.baidu.com',
  port: '8080'
};
Object.freeze(INFO);
INFO.port = '443'; //Cannot assign to read only property
console.log(INFO);

# 传值与传址

基本数据类型指数值、字符串等简单数据类型，引用类型指对象数据类型。

基本类型复制是值的复制，互相不受影响。

对于引用类型来讲，变量保存的是引用对象的指针。变量间赋值时其实赋值是变量的指针，这样多个变量就引用的是同一个对象。

# undefined

对声明但未赋值的变量返回类型为 undefined 表示值未定义。

对未声明的变量使用会报错，但判断类型将显示 undefined。

函数参数不传或者没返回值也是undefined

# null

用于定义一个空对象，即如果变量要用来保存引用类型，可以在初始化时将其设置为null

# 严格模式

严格模式可以让我们及早发现错误，使代码更安全规范，推荐在代码中一直保持严格模式运行。

"use strict";
// 变量必须使用关键词声明，未声明的变量不允许赋值, 强制声明防止污染全局
function run() {
  web = "jenny"; // web is not defined
}
run();
console.log(web); // jenny

// 关键词不允许做变量使用
var public = 'jenny'
// 变量参数不允许重复定义
unction hd(name, name) {} 

// 可单独为函数设置严格模式
function strict(){  
  "use strict";  
  return "严格模式";  
}  
function notStrict() {  
  return "正常模式";  
}  
// 严格模式下必须使用声明
{name,url} = {name:'Jenny',url:'baidu.com'} /// name is not defined
