# 数据类型

# JS的数据类型

js数据类型：基本据类型和引用数据类型。 基本数据类型有：String、Number、boolean、null、undefined 引用数据类型有：object（里面包含的 function、Array、Date）

# 数据类型判断方式

  1. typeof 一般用来判断基本数据类型，不能精确判断引用类型，输出分别是：
console.log(typeof 2);               // number
console.log(typeof NAN);               // number  NAN 是Number 中的特殊数值
console.log(typeof true);            // boolean
console.log(typeof 'str');           // string
console.log(typeof undefined);       // undefined
console.log(typeof []);              // object     []数组的数据类型在 typeof 中被解释为 object
console.log(typeof {});              // object
console.log(typeof null);            // object     null 的数据类型被 typeof 解释为 object
console.log(typeof function(){});    // function

  • typeof对数字Number，布尔值Boolean，字符串String，函数Function，对象Object，Undefined的类型判断都是精确的
  • 引用类型，除 function 以外，一律返回 object 类型
  • null 有属于自己的数据类型 Null ， 但被 typeof 解释为 object
  • 引用类型中的 数组、日期、正则 也都有属于自己的具体类型，而 typeof 对于这些类型的处理，只返回了处于其原型链最顶端的 Object 类型。

NAN 是Number 中的特殊数值，js 规定的NaN 不等于NaN 用 isNaN（） 检测是否是非数值型， 不是数字返回true

Number('as') == NaN  // false
NaN == NaN  // false

  1. instanceof 用于检测构造函数的 prototype， 属性是否出现在某个实例对象的原型链上， 一般用于引用数据类型的判断，检测基本类型不一定准确
    注意： instanceof 不能用于判断 null 和 undefined instanceof 只能用来判断两个对象是否属于实例关系， 而不能判断一个对象实例具体属于哪种类型。 instanceof对于引用类型的支持很好，所以一般都是在typeof判断为object时才使用instanceof
function Car(make, model, year) {
  this.make = make;
  this.model = model;
  this.year = year;
}
const auto = new Car('Honda', 'Accord', 1998);
console.log(auto instanceof Car) // true
console.log(auto instanceof Object)  // true

  2. constructor 返回创建该对象的构造函数 undefined和null没有contructor属性
// 字符串：String()
let str = 'aaa'
alert(str.constructor); // function String() { [native code] }
alert(str.constructor === String); // true
console.log(new String('123') instanceof String);  //true
console.log('123' instanceof String);  //false
 
// 数字：Number()
let num = 1
alert(num.constructor); // function Number() { [native code] }
alert(num.constructor === Number); // true

// 数组：Array()
var arr = [1, 2, 3];
alert(arr.constructor); // function Array() { [native code] }
alert(arr.constructor === Array); // true

// 日期 ： Date()
let date = new Date();
alert(date.constructor) //function Date() { [native code] }
alert(date.constructor === Date) //  true

// 对象：Object()
let o = { name : "张三"};
alert(o.constructor); // function Object() { [native code] }
alert(o.constructor === Object); // true

// 函数： Function()
let func = function (){alert('CodePlayer');};
alert(func.constructor) //function Function() { [native code] }
alert(func.constructor === Function) // true
alert(func.prototype.constructor); // function(){ alert("CodePlayer"); }

  • 但JS对象的constructor是不稳定的，这个主要体现在自定义对象上，当开发者重写prototype后，原有的constructor会丢失，constructor会默认为Object
  1. Object.prototype.toString.call() 精确判断数据类型， 该方法默认返回其调用者的具体类型
console.log(Object.prototype.toString.call(123));    //[object Number]
console.log(Object.prototype.toString.call('123'));    //[object String]
console.log(Object.prototype.toString.call(undefined));    //[object Undefined]
console.log(Object.prototype.toString.call(true));    //[object Boolean]
console.log(Object.prototype.toString.call({}));    //[object Object]
console.log(Object.prototype.toString.call([]));    //[object Array]
console.log(Object.prototype.toString.call(function(){}));    //[object Function]
console.log(Object.prototype.toString.call(null));    //[[object Null]]
console.log(Object.prototype.toString.call(newDate());// [object Date]
console.log(Object.prototype.toString.call(new RegExp());// [object RegExp]
console.log(Object.prototype.toString.call(new Error());// [object Error]
console.log(Object.prototype.toString.call(document));// [object HTMLDocument]
console.log(Object.prototype.toString.call(window));//[object global] window 是全局对象 global 的引用

// 精准获取数据类型封装
function isType (data) {
  if (data === '') return '';

  let istypeArray = ['Object', 'Number', 'Array', 'Boolean', 'Null', 'Undefined', 'String', 'Symbol', 'Date', 'BigInt', 'Function'];
  let is = Object.prototype.toString.call(data);
  for (let i = 0; i < istypeArray.length; i++)
  {
    if (is.includes(istypeArray[i])) return istypeArray[i]
  }
}

js中的对象都继承自Object，当我们在某个对象上调用一个方法时，会先在该对象上进行查找，如果没找到则会进入对象的原型（也就是.prototype）进行查找，如果没找到，同样的也会进入对象原型的原型进行查找，直到找到或者进入原型链的顶端Object.prototype才会停止。
