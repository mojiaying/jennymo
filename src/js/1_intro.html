# JS 简介

JavaScript 官方名称是 ECMAScript 是一种属于网络的脚本语言,已经被广泛用于Web应用开发,常用来为网页添加各式各样的动态功能,为用户提供更流畅美观的浏览效果。

1995年2月Netscape的布兰登.艾奇开发了针对网景公司的 Netscape Navigator浏览器的脚本语言LiveScript。之后Netscape与Sun公司联盟后LiveScript更名为JavaScript。

ECMA是欧洲计算机制造商协会由Sum、微软、NetScape公司的程序员组成。

# 使用场景：

  1. 浏览器网页端开发
  2. 做为服务器后台语言使用Node.js(opens new window)
  3. 移动端手机APP开发，如Facebook的 React Native (opens new window)、uniapp、PhoneGap、IONIC
  4. 跨平台的桌面应用程序，如使用 electronjs

# 发展历史

  1. 1994年Netscape（网景）公司发布了 Navigator 浏览器1.0版本，市场占用率超过90%

  2. 1995年发布了JavaScript 语言

  3. 1996年 JS在 Navigator浏览器中使用

  4. 1996年微软发布JScript在IE3.0中使用

  5. 1996年11月网景公司将JS提交给ECMA(国际标准化组织)成为国际标准，用于对抗微软。

  6. 由ECMA的第39号技术专家委员会（Technical Committee 39，简称TC39）负责制订ECMAScript标准，成员包括Microsoft、Mozilla、Google等大公司。

  7. 1997年 ECMA发布ECMA-262标准，推出浏览器标准语言 ECMAScript 1.0

  8. ECMAScript 是标准而Javascript是实现

...

  1. 2009年ECMAScript 5.0发布

  2. 2011年ECMAScript5.1发布，成为ISO国际标准，从而推动所有浏览器都支持

...

  1. 2015年ECMAScript6发布，更名为ECMAScript 2015。

  2. 2016年ECMAScript7发布，ECMAScript2016

  3. 2017年ECMAScript8发布，ECMAScript2017

  4. 2018年ECMAScript9发布，ECMAScript2018

  5. 2019年ECMAScript10，ECMAScript2019

  6. 2020年ECMAScript11，ECMAScript2020

....

从2015年开始 tc39委员会决定每年发布新的ECMAScript版本

# 调试工具

Mac: option + command + J
win: F12 , ctrl + shift + I

# 运行流程

所有内容需要在特定的环境中运行，就像PSD需要在类似PS的软件处理一样。浏览器内置了处理的JS的解析器，但不同浏览器的性能不同，所以JS一般都在浏览器中执行，当然也有可以在服务器后台执行的JS解析器。

# 脚本定义

  1. 像style标签一样，可以在html文档中使用script标签嵌入javascript代码。
  2. 通过设置 src 属性引入外部js文件。

# 加载延时原因：

如果js放在 <head>标签中要等到js加载并解析后才会显示<body>标签中的内容。 推荐做法：可以将js放在 body里面最后

# 注释

单行注释： // 注释内容.....
多行注视： /*注释内容.....*/

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>title</title>
</head>
<body>
    <h1>xxxxxxxxxx</h1>
    <script src="index.js"></script>
    <script>
      // 注释内容.....
      /*
      注释内容.....
      注释内容.....
      */
    </script>
</body>
