# 常见问题处理

# 修改commit信息

# 一、 修改上一条（已经push或者还没push的）：

  1. git commit --amend出现编辑界面
  2. 按 i 键，可以编辑内容
  3. 修改完后按esc键退出编辑，输入:wq保存并推出

# 二、修改历史push的commit信息：

方法一：指定修改条数

  1. git rebase -i HEAD~n
  2. 按 i 键，可以编辑内容
  3. 将要修改的commit信息ID前的pick改成edit
  4. 修改完后按esc键退出编辑，输入:wq保存并推出
  5. git log查看选中的已变成最新的提交
  6. 输入git commit --amend，编辑修改保存退出
  7. git rebase --continue 继续下一个修改
  8. git push
  9. 退出Rebase状态git rebase --abort

方法二：使用commitId

  1. git rebase -i {commitId}
  2. 。。。后面一样

# git 几种修改选择

  • pick: 保留该commit
  • reword: 保留该commit, 但可以修改commit的massage
  • edit: 保留该commit，但可以修改提交，包括message和提交文件
  • squash: 将该commit和前一个commit合并
  • fixup: 将该commit和前一个commit合并, 不保留该提交的注释信息
  • drop: 丢弃这个commit

# git配置的用户名、密码

git 报 HTTP Basic: Access denied 错误 本地git配置的用户名、密码与gitlabs上注册的用户名、密码不一致。

  1. git config –system –unset credential.helper 重新输入账号密码
  2. 如果不行就输入：git config –global http.emptyAuth true

# 直接修改本地git配置的用户名、密码
