  • 计算属性的 setter 计算属性名不是方法名，计算属性里面有get 和 set两个方法，属性值默认取 get的返回值，所以使用计算属性时不需要()。 当运行vm.reversedMessage = "xxx"时会调用set方法，一般不使用。
computed: {
  fullName: {
    // getter
    get: function () {
      return this.firstName + ' ' + this.lastName
    },
    // setter
    set: function (newValue) {
      var names = newValue.split(' ')
      this.firstName = names[0]
      this.lastName = names[names.length - 1]
    }
  }
}

# 监听属性watch

  • 当需要在数据变化时执行异步或开销较大的操作时可使用watch
  • watch 选项允许我们执行异步操作 (访问一个 API)，限制我们执行该操作的频率，并在我们得到最终结果前，设置中间状态。这些都是计算属性无法做到的。

# computed VS watch, methods

  • computed
    • 不支持异步
    • 会进行缓存，如果多次使用时，计算属性只会调用一次。
    • 只有依赖数据发生改变，才会重新进行计算，一个多对一或者一对一
    • computed属性不需要在data中声明
  • watch
    • 支持异步
    • 不支持缓存，数据变，直接会触发相应的操作
    • 数接收两个参数，监听属性的新值和旧值
    • 被监听属性发生变化时，需要执行对应的操作，一对多
    • 被监听属性必须是data中声明过或者父组件传递过来的props中的数据
    • 如果监听属性是对象或数组时，旧值将与新值相同，因为它们的引用指向同一个对象/数组。Vue 不会保留变更之前值的副本。
  • methods
    • 要命令式触发，方法每次触发都会重新计算。

# 事件监听

  • 事件监听使用 v-on 指令， 缩写： @
  • v-on调用方法传参
    1. 没有传参数时，方法使用可以省略括号
    2. 只有一个参数时
      • 省略括号，vue会默认将浏览器事件作为参数传进来
      • 写了括号不传参，参数值为undefined
    3. 有多个参数时
      • 如果省略了括号，vue会默认将浏览器事件作为第一个参数传进来后面的参数则为undefined
      • 写了括号不传参，所以参数值都为undefined
      • $event 可以直接获取到浏览器事件对象
  • v-on修饰符
    • .stop-调用 event.stopPropagation()阻止冒泡
    • .prevent - 调用 event.preventDefault()阻止默认事件
    • .{keyCode | keyAlias}- 只当事件是从特定键触发时才触发回调。
    • .once - 只触发一次回调
    • .native- 监听组件跟元素的原生事件
  <!-- 阻止单击事件继续传播 -->
<a v-on:click.stop="doThis"></a>

<!-- 提交事件不再重载页面 -->
<form v-on:submit.prevent="onSubmit"></form>

<!-- 修饰符可以串联 -->
<a v-on:click.stop.prevent="doThat"></a>

<!-- 只有修饰符 -->
<form v-on:submit.prevent></form>

<!-- 添加事件监听器时使用事件捕获模式 -->
<!-- 即内部元素触发的事件先在此处理，然后才交由内部元素进行处理 -->
<div v-on:click.capture="doThis">...</div>

<!-- 只当在 event.target 是当前元素自身时触发处理函数 -->
<!-- 即事件不是从内部元素触发的 -->
<div v-on:click.self="doThat">...</div>
<!-- 按回车键触发事件 -->
<button @keyup.enter="doSomething">会车键触发事件</button>
<button @keyup.13="doSomething">会车键触发事件</button>

# v-if和v-show

  • v-if: 当条件为false时压根不会有对应的元素在DOM中,不需要频繁切换时使用
  • v-show: DOM始终会渲染出来，本质上是display控制是否要显示,内容需要频繁切换时使用

# v-for 中 key属性的作用

v-for 渲染的元素列表时，它默认使用“就地更新”的策略。如果数据项的顺序被改变，Vue 将不会移动 DOM 元素来匹配数据项的顺序，而是就地更新每个元素内容，并且确保它们在每个索引位置正确渲染。 例如：在B和C之间加一个F

vue 会把C更新成F，D更新成C，E更新成D，最后再插入E
为了给 Vue 一个提示，以便它能跟踪每个节点的身份，从而重用和重新排序现有元素，你需要为每项提供一个唯一 key，key的作用主要是为了高效的更新虚拟DOM

# 检测数组更新

Vue中包含了一组观察数组编译的方法，使用它们改变数组也会触发视图的更新

  • push()
  • pop()
  • shift()
  • unshift()
  • splice()
  • sort()
  • reverse() 也有非变更方法，例如 filter()、concat() 和 slice()，它们不会变更原始数组，而总是返回一个新数组。可以直接替换数组来触发视图更新
example1.items = example1.items.filter(function (item) {

# 表单输入绑定

  • v-model
    • <input type="text" v-model="msg"/>
    • 本质 :value 和 @input 的结合
      <input type="text" :value="msg" @input="msg = $event.target.value"/>
  • 不同类型的表单绑定
<!-- 多行文本 -->
<textarea v-model="message" placeholder="add multiple lines"></textarea>
<!-- 单选 -->
<label for="male"> <input type="radio" value="男" v-model="sex">男 </label>
<label for="female"> <input type="radio"  value="女" v-model="sex">女 </label>
<!-- 单个复选框值为布尔值 -->
<label> <input type="checkbox"  v-model="isAgree"></label>
<!-- 多个复选框对应的是元素的value值： -->
<input type="checkbox" value="篮球" v-model="hobbies">篮球
<input type="checkbox" value="足球" v-model="hobbies"> 足球
<!-- 单选下拉值为字符串 -->
<select name="abc" v-model="fruit">
    <option value="苹果">苹果</option>
    <option value="香蕉">香蕉</option>
    <option value="榴莲">榴莲</option>
    <option value="葡萄">葡萄</option>
</select>
<!-- 多选下拉值为数组类型 -->
<select name="abc" v-model="fruits" multiple>
  <option value="苹果">苹果</option>
  <option value="香蕉">香蕉</option>
  <option value="榴莲">榴莲</option>
  <option value="葡萄">葡萄</option>
</select>
<!-- 使用列表渲染多个复选框 -->
<label v-for=“item in hobbiesList” :for='item'>
  <input type=“checkbox” :value='item' :id='item' v-model='hobbies'> {{item}}
</label>
<!-- 列表渲染下拉选择框 -->
<select v-model="selected">
  <option v-for="option in options" v-bind:value="option.value">
    {{ option.text }}
  </option>
</select>
