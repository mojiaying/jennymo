# 数字精度丢失问题

# JS 数字丢失精度的原因

计算机的二进制实现和位数限制有些数无法有限表示。就像一些无理数不能有限表示, JS 遵循 IEEE 754 规范，采用双精度存储（double precision），占用 64 bit

  1. 两个简单的浮点数相加 比如 0.1+0.2 不等于 0.3 0.1 >> 0.0001 1001 1001 1001…（1001无限循环）
    0.2 >> 0.0011 0011 0011 0011…（0011无限循环）

  2. 整数最大范围 整数是按最大54位来算最大(253 - 1，Number.MAX_SAFE_INTEGER,9007199254740991) 和最小(-(253 - 1)，Number.MIN_SAFE_INTEGER,-9007199254740991) 安全整数范围的。所以只要超过这个范围，就会存在被舍去的精度问题。

  3. toFixed 不会四舍五入

# 解决方案

对于整数，前端出现问题的几率可能比较低，毕竟很少有业务需要需要用到超大整数，只要运算结果不超过 Math.pow(2, 53) 就不会丢失精度。
对于小数，前端出现问题的几率还是很多的，尤其在一些电商网站涉及到金额等数据。解决方式：把小数放到位整数（乘倍数），再缩小回原来倍数（除倍数） (0.1*10 + 0.2*10) / 10 == 0.3 // true toFixed的修复如下

// toFixed 修复
function toFixed(num, s) {
    var times = Math.pow(10, s)
    var des = num * times + 0.5
    des = parseInt(des, 10) / times
    return des + ''
}

# 深拷贝，浅拷贝

  1. 浅拷贝: 复制对象的引用，跟原对象共享内存，Object.assign()/Object.create(obj1)/arrObj.slice(start, end) / arrObj.concat() 只拷贝一层属性
  2. 深拷贝： 复制对象每一个属性和值，跟原对象不共享内存
    1. 使用JSON.stringify+JSON.parse实现深拷贝
      1. 它会抛弃对象的constructor，深拷贝之后，不管这个对象原来的构造函数是什么，在深拷贝之后都会变成Object；
      2. 这种方法能正确处理的对象只有 Number, String, Boolean, Array, 扁平对象，像function没办法转成JSON；
    2. jquery 提供一个$.extend可以用来做深拷贝；var obj2 = $.extend(true, {}, obj1);
    3. 递归深拷贝:
    function deepCopy(obj1) {
  //  创建一个空的对象或数组
  var obj2 = Array.isArray(obj1) ? [] : {};
  if (obj1 && typeof obj1 === "object") {
    for (var i in obj1) {
      let prop = obj1[i]; 
      // 避免相互引用造成死循环，如obj1.a=obj
      if (prop == obj1) {
        continue;
      }
      if (obj1.hasOwnProperty(i)) {
        // 如果子属性为引用数据类型，递归复制
        if (prop && typeof prop === "object") {
          obj2[i] = deepCopy(prop);
        } else {
          // 如果是基本数据类型，只是简单的复制
          obj2[i] = prop;
        }
      }
    }
  }
  return obj2;
}

# 判断两个对象是否相等

  1. Object.is(a, b) a和b是不是同一个指针的对象
  2. 判断两个对象内容是否相同
    function isObjectValueEqual(a, b) {
  // 1.用Object.getOwnPropertyNames拿到对象的所以键名数组
    var aProps = Object.getOwnPropertyNames(a);
    var bProps = Object.getOwnPropertyNames(b);
  // 比对键名数组的长度是否相等
    if (aProps.length != bProps.length) {
        return false;
    }
  // 比对键名对应的键值是否相等
    for (var i = 0; i < aProps.length; i++) {
        var propName = aProps[i];
        var propA = a[propName];
        var propB = b[propName];
        if ( propA !== propB) {
                return false;
        }
    }
    return true;
}

# cookie , localstorage, sessionstorage , vuex 区别

  1. cookie 由服务器生成，可以设置失效，浏览器生成的cookies默认关闭浏览器会失效，4K左右，与服务器通讯，携带在请求头中，一般登录唯一标识符号存在cookies里
    1. 创建document.cookie="username=John Doe";
    2. 添加过期时间和存储路径document.cookie="username=John Doe; expires=Thu, 18 Dec 2043 12:00:00 GMT";path=/
    3. 读取：document.cookie将以字符串的方式返回所有的 cookie，分号隔开cookie1=value; cookie2=value; cookie3=value;
  2. sessionstorage 刷新页面后还存在，但关闭页面后会清空，5MB左右，不参与通讯
  3. localstorage除非手动删除，否则永久保存，不参与通讯
  4. localStorage和sessionStorage都具有相同的操作方法，例如setItem(key, val)、getItem(key)和removeItem(key),clear清除所有的key/value等
  5. 其他操作方法(./[])：var storage = window.localStorage; storage.key1 = "hello"; storage["key2"] = "world";
  6. 遍历本地存储：
  var storage = window.localStorage;
  for(var i=0, len=storage.length; i<len;i++){
      var key = storage.key(i);     
      var value = storage.getItem(key);     
      console.log(key + "=" + value); 
  }

# 防抖（Debouncing）

函数防抖是指对于在事件被触发n秒后再执行的回调，如果在这n秒内又重新被触发，则重新开始计时 这样可以把多个顺序地调用合并成一次， 函数只会在最后一次触发后执行一次，适用于监听屏幕滚动，鼠标移动 防止多次点击导致表单多次提交 思路：用 setTimeout 实现计时，配合 clearTimeout 实现“重新开始计时”。
初步实现：

function debounce(func,wait) {
    let timer = null;
    return function () {
        clearTimeout(timer);
        timer = setTimeout(func,wait);
    }
}

# 使用事例

container.onmousemove = debounce(doSomething, 1000);
1

每当事件被触发，执行的都是那个被返回的闭包函数。
因为闭包带来的其作用域链中引用的上层函数变量声明周期延长的效果，debounce 函数的 settimeout计时器 ID timeout 变量可以在debounce 函数执行结束后依然留存在内存中，供闭包使用。

注意：通过返回函数调用，每一次调用返回的是不同的函数，绑定事件无法移除

# 对比未做防抖

container.onmousemove = doSomething
1

对比做了发抖之后，doSomething 函数的 this 变成了闭包匿名函数的 this ，前者变成了指向全局变量。 同理，doSomething 函数参数也接收不到 MouseEvent 事件了。

# 修复this指向

修复this指向目标函数可以获取绑定的事件对象

function debounce(func, wait) {
    let timeout
    return function () {
        let self = this //传给目标函数
        clearTimeout(timeout)
        timeout = setTimeout(()=>{func.apply(self, arguments)}, wait)
    }
}

# 优化：立即执行

我们有时候希望能实现 在一个周期内第一次触发，就立即执行一次，然后一定时间段内都不能再执行目标函数。
这样，在限制函数频繁执行的同时，可以减少用户等待反馈的时间，提升用户体验。

function debounce(func, wait, immediate) {
    let timeout
    let debounced = function() {
        let self = this
        if(timeout) clearTimeout(timeout)
        if(immediate) {
            let callNow = !timeout
            if(callNow) func.apply(self, arguments)
            timeout = setTimeout(
                ()=>{time = null} //见注解
            , wait)
        } else {
            timeout = setTimeout(
                ()=>{func.apply(self, arguments)}
            , wait) 
        }
    }
    return debounced
}

  1. clearTimeout(timeout)清除了timeout指向的定时器
  2. timeout=null，是修改timer的指向，是timeout这个变量不指向某个定时器，然而并没有清除这个定时器，定时器依旧可以使用
  3. timeout作为开关变量，timeout=null表明一个周期结束。使得 callNow 为 true，目标函数可以在新的周期里被触发时被执行
  4. timeout 作为闭包引用的上层函数的变量，是不会自动回收的。手动将其设置为 null ，让它脱离执行环境，一边垃圾收集器下次运行是将其回收。

# 优化：取消立即执行

函数也是对象，也可以为其添加属性。 为了添加 “取消立即执行”功能，为 debounced 函数添加了个 cancel 属性，属性值是一个函数

debounced.cancel = function() {
        clearTimeout(time)
        time = null
 }

完整代码：

function debounce(func, wait, immediate) {
    let time
    let debounced = function() {
        let self = this
        if(time) clearTimeout(time)
        if(immediate) {
            let callNow = !time
            if(callNow) func.apply(self, arguments)
            time = setTimeout(
                ()=>{time = null}
            , wait)
        } else {
            time = setTimeout(
                ()=>{func.apply(self, arguments)}
            , wait) 
        }
    }

    debounced.cancel = function() {
        clearTimeout(time)
        time = null
    }

    return debounced
}

let isClick = false

方法二：绑定返回函数计算两次执行时间差

function throttling() {
  let preTime = 0
  return () => {
    let nowTime = new Date().getTime();  //频繁操作都会执行此部事件
    if( nowTime - preTime > 1000 ){   // 只有间隔1s之后，才输出一次结果
      console.log('执行事件');
      preTime =  new Date().getTime();  // 更新历史时间
    }
  }
}

# 滚动到底部监听

function scroll() {
  // 可滚动高度, 即文档高度
  let scrollHeight = document.documentElement.offsetHeight
  // 可视区域高度
  let viewHeight = document.documentElement.clientHeight
  // 已滚动高度
  let scrollTop = document.documentElement.scrollTop
  // 判断是否触底
  if(scrollTop >= scrollHeight - viewHeight) {
    // dosomething
  }
}
