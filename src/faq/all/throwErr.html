账号绑定正常 2. Permission denied (publickey) 没有权限的publickey ，出现这错误一般是以下两种原因

  • 客户端与服务端未生成 ssh key
  • 客户端与服务端的ssh key不匹配 解决办法: 重新生成一次ssh key ，服务端也重新配置一次
    • 生成ssh密钥ssh-keygen -t rsa -C "youremail@example.com"
    • 查看生成的密钥cat ~/.ssh/id_rsa.pub

// 找到config文件，并且编辑config文件

# Gitee(码云)、Github同时配置ssh key

  1. 把现有的ssh删掉rm -rf ~/.ssh/*(这句命令行如果你多打一个空格，可能就要重装系统了，建议复制运行)
  2. 删除后cat ~/.ssh/id_rsa.pub找不到密钥
cat ~/.ssh/id_rsa.pub
cat: /Users/jennymo/.ssh/id_rsa.pub: No such file or directory
1
2
  1. 依次生成两个平台的key
$ ssh-keygen -t rsa -C "xxxxxxx@qq.com" -f "github_id_rsa"
$ ssh-keygen -t rsa -C "xxxxxxx@qq.com" -f "gitee_id_rsa"
1
2
  1. 依次给两个host添加对应的ssh密钥文件 在.ssh文件夹下执行命令vi config 文件中添加以下内容
# gitee
Host gitee.com
HostName gitee.com
PreferredAuthentications publickey
IdentityFile ~/gitee_id_rsa

# github
Host github.com
HostName github.com
PreferredAuthentications publickey
IdentityFile ~/github_id_rsa
