# nvm 基本使用

# nvm，node，npm区别

nvm：nodejs 版本管理工具。 也就是说：一个 nvm 可以管理很多 node 版本和 npm 版本。 nodejs：在项目开发时的所需要的代码库 npm：nodejs 包管理工具。 在安装的 nodejs 的时候，npm 也会跟着一起安装，它是包管理工具。 npm 管理 nodejs 中的第三方插件

# nvm、nodejs、npm关系

nvm 管理 nodejs 和 npm 的版本。npm 可以管理 nodejs 的第三方插件。

# nvm 常用命令

nvm ls-remote  ## 查看远程服务器上所以可用node版本
nvm ls ## 查看本地安装node版本列表
nvm i [version No.] ## 安装指定版本
nvm current ## 查看当前版本
nvm use [version No.]  ## 切换指定node版本
nvm --version ## 查看nvm当前版本
nvm alias [name] [version No.] ## 给不同的版本号添加别名
nvm use [name] ## 通过别名使用指定版本
nvm unalias [name] ## 删除已定义的别名
nvm reinstall-packages [version No.] ## 在当前版本node环境下，重新全局安装指定版本号的npm包
nvm uninstall [version No.] ## 卸载制定的版本
nvm use node ## 切换最新版本
nvm run 4.2.2 --version ## 直接运行特定版本的node
nvm exec 4.2.2 node --version ## 在当前终端的子进程中运行特定版本的 Node
nvm which 4.2.2 ## 确认某个版本Node的路径
nvm install unstable ## 安装最新不稳定版本的 Node
